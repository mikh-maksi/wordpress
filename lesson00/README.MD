# WordPress
## Причина створення CMS
CMS (системи управління контентом) дозволяють економити час кваліфікованих спеціалістів (програмстів) на створення та адміністрування Інтернет-ресурсів.  Станадартною функціональністю CMS є можливість адміністрування сайту без знаання програмування.  Проте, через те, що більшість сайтів для малого бізнесу є схожими за функціональністю були створеня типові рішення, які доступні для використання в рамках екосистеми відповідної CMS.  
Наявність готових функціональних (плагіни) та графічних (теми) блоків дозволяють значно пришвидшувати розробку простої функціональності, проте, при збільшенні складності архітектурні обмеження CMS обмежуть гнучкість розробки і максимальну швидкодію, яка доступна для відповідної CMS.  
Тобто для кожного типу сайтів є певні вимоги, які можуть забезпечуватися різними типами CMS. І, за умови їх виконання, критичних складностій із використанням CMS не виникає.  Навпаки, можливості CMS дозволяють швидко перевірити бізнес-припущення, отримати оплату за товари чи послуг, тим самим окупити витрати на побудову цифрової інфраструктури і мати обгрнтування, щодо вкладання грошей в розробку більш складних інформаціних систем.  


## Чому CMS WordPress має найбільшу долю ринку
WordPress Виграв конкурентну боротьбу із іншими CMS. Один із найбільш успішних рішень є те, що всі елементи можна встановити безпосередньо на самому сайті (CMS скачує потрібні теми та плагіни із серверу), тоді як у конкуртентів було потрібно спочатку скачати відповідні елементи з серверу і далі завантажити їх на сайт (CMS Joomla, CMS Drupal).  Також повпливало те, що при переході на нову версію CMS Joomla розробники не забезпечили зворотньої підтримки старих версій, при цьому нові були не стабільні (в той час лідерство остатночно перейшо до WordPress). В результаті - CMS WordPress є де-факто стандартом для створення сайтів під управлінням CMS. При цьому це може бути як он-лайн ЗМІ (WordPress первинно створювався як рушій для блогу) так і Інтернет-магазин (плагін WooCommerce), власна соціальна мережа (плагін BudyPress) та багато чого іншого.

## Послідовість дій з використання WordPress
Перший крок - це встановлення ядра WordPress. Для великої кількості популяних хостингів існує функція автоматичного встановлення. Після встановлення - потрібно обрати тему. І, дуже часто, зручним варіантом є встановлення демо-контенту для теми. В подальшому - формування прототипу, із відштовхуванням від демо-контенту. І, за необхіності створення теми з нуля, на базі унікального дизайну, створеного дизайнером на замовлення.  
Щодо додавання функціональних елементів, то для початкового рівня функцій достатньо безкоштовни плагівнів. Частина плагінів пропонує спеціальні (додаткові) функції із доплатою. Але, в певних обставинах, можуть знадобитися плагіни спеціально розроблені під виряшення спеціальної задачі.
